Branching

Branching allows teams of developers to easily collaborate inside of one central code base. When a developer creates a branch, the version control system creates a copy of the code base at that point in time. Changes to the branch don't affect other developers on the team.

*Each repository has one default branch*

Merge Files

To merge files in GitHub you typically follow these steps:

  1. Create a New Branch: create a new branch for your changes:
    git checkout -b <new-branch-name>
  2. Make Changes: Edit or add the files you want to merge.
  3. Stage Your Changes: Add the changes to the staging area:
    git add <file1> <file2> ...
  4. Commit Your Changes: Commit the staged changes with a message:
    git commit -m "Describe your changes"
  5. Push Your Branch: Push your branch to the remote repository:
    git push origin <new-branch-name>
  6. Create a Pull Request: Go to the GitHub repository in your web browser. You'll see an option to create a pull request for your newly pushed branch.
  7. Review and Merge: After reviewing, you can merge the pull request. You may need to resolve any merge conflicts if there are changes on the base branch.
  8. Delete the Branch: Once merged, you can delete the branch both locally and remotely:
    git branch -d <new-branch-name>
    git push origin --delete <new-branch-name>

That's it! Your files are now merged in the repository.
