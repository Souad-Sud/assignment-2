The Git Commands to Link the Local and Remote Repos

Step 1: Create a new directory.

mkdir (directory name)

This command creates a new directory name (example:mkdir git-project)

Step 2:Navigate to the project directory.

cd /path/to/your/project-directory

This command is used for to navigate to the project

example: cd ~/documents/git-project

Step 3. Initialize a local repository(if not already intilized)

git init

This command is used to Initialize a local Git repository.

Step 4: Create a new file

touch filename.txt

This command is used for creates a new file

Step 5: Check the status of your repository

git status

This command is used to check the status of your repository

Step 6: Add the file to the staging area

git add (filename) or git add .

This command is used for Add the file to the staging area

Step 7: Make a Local commit

git commit -m "intial commit"

This command is used for creates a local commit

Step 8: Add the remote repository URL

git remote add origin <remote-repo-URL>

This command used for Add the remote repository URL

Step 9. Verify the remote repository

git remote -v

This command is used for verify the remote repository

Step 10. Push local code to the remote repository

git push -u origin master

This command is used for to Push the local code to the remote repository

======= Github Project 101

Git Commands for Creating a Local Project

1. Initialize a Git Repository

git init

Initialize an empty Git repository in the current directory.

2. Clone a Remote Repository

git clone <repository-url>

Clone a remote repository to your local machine.

3. Check Status

git status

Check the status of your working directory and staging area.

4. Stage Files

git add <file>

Stage specific files or all files (`git add .`) for commit.

5. Commit Changes

git commit -m "Commit message"

Create a commit with the staged changes and a descriptive message.

6. Create a Branch

git checkout -b <branch-name>

Create a new branch and switch to it.

Contribution to a GitHub Project

A step by step guide on how to contribute

Project Overview

This project is open-source and welcomes contributions from developers worldwide. The goal is to build a robust, scalable web application for handling task management.

We use modern technologies like HTML, CSS, JavaScript, and Node.js. Whether you're an experienced developer or a beginner, you're welcome to join!

How to contribute

  1. Fork the reporsitory from GitHub
  2. Clone your forked reporsitory locally 
     git clone:https://github.com/your-username/repo-name.git
  3. Make your changes and commit them: 
    git add .
    git commit -m "Describe your changes"
  4. Push your branch back to your GitHub fork: 
    git push origin feature-or-bugfix-name
  5. Create a pull request (PR) on the original repository.

Contribution Guidelines

  • Follow our Code of Conduct when interacting with other contributors.
  • Ensure your code is clean, with meaningful comments and documentation.
  • Make sure your feature or bugfix includes relevant unit tests.
  • Always check for existing issues before submitting a new one.
